# Commit Guide

Commit messages are generated from:

- Changed files
- Knowledge (implementation notes, ADRs, conventions)
- Current phase and artifacts
- Review and QA findings

Conventional types: feat, fix, refactor, docs, test, perf, build, ci, style, chore, revert.

**EM approval required** before commit:

```bash
engineeringos repo approve commit
engineeringos repo commit
```
