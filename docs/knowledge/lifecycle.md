# Knowledge Lifecycle

1. **Capture** — from events, employees, or explicit API calls
2. **Classification** — type and scope assignment
3. **Validation** — source, confidence, duplication, conflicts
4. **Promotion** — Engineering Manager approves upward flow
5. **Retrieval** — contextual, relevance-scored
6. **Relationships** — graph edges (depends_on, supersedes, etc.)
7. **Versioning** — content updates with audit trail
8. **Deprecation / Archival / Retirement** — aging policies

Conversation knowledge auto-archives after 30 days. Stale drafts deprecate after 180 days.
